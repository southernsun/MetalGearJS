Audio storage in the
Metal Gear ROM

How the MSX2 release stores its music and sound effects — not as recordings, but as compact streams of commands that drive the sound chip in real time. Several examples below are synthesized live in the browser from the original ROM bytes.

HARDWAREThe constraint: a 3-voice synthesis chip

The MSX has no sampled-audio hardware. All sound is produced by a Programmable Sound Generator (a General Instrument AY-3-8910), which can only generate:

  • 3 square-wave tone channels (A, B, C), each with an independent 12-bit period and 16 volume levels;
  • 1 noise generator (5-bit period) that can be mixed into any channel;
  • 1 hardware volume envelope shared across channels.

The chip is controlled entirely by writing 14 registers. There is no way to store a waveform — only periods and volumes. Music and effects must therefore be expressed as sequences of register writes over time.

PSG register map

Higher pitch = smaller period. Period and volume are all the chip stores.

DRIVERA 60 Hz interpreter over byte streams

A small driver runs once per video frame (60 Hz, on the vertical-blank interrupt). It maintains four channels — three for music, one for sound effects — and on every frame advances each channel through its own byte stream, writing PSG registers.

Each channel stream is a program in a compact bytecode: one byte usually encodes a whole musical event (a pitch and a duration); other bytes set tempo, octave, instrument, or jump within the stream. The driver is the executor; the byte stream is the instruction set.

This separation is why the entire soundtrack is tiny: roughly 5 KB of bytecode encodes about ten tracks, with sound effects in two further small regions.

Per channel, per frame

stepaction
1decrement the note's frame counter
2counter > 0 → update envelope / vibrato / instrument tick
3counter = 0 → fetch & execute the next command byte(s)
4write the resulting period & volume to the PSG

A 32-byte work area per channel holds the stream pointer, note counter, octave, tempo, envelope state, loop/return addresses, etc.

PITCHFrom a nibble to a frequency

The pitch nibble (0–11) indexes a table of base periods. The current octave then left-shifts that period; the chip frequency is clock / (16 × period) with a ≈1.79 MHz clock.

base period[0..11] = 6B 65 5F 5A 55 50 4C 47 43 40 3C 39

Because the octave multiplies the period, a larger octave index produces a lower pitch: index 0 ≈ 1045 Hz, and each step up the octave value halves it.

Duration is tempo × (index + 1) frames (or just tempo when the index is 0), at 60 frames per second.

TIMBREInstruments: a note becomes an envelope

A bare square wave is thin. An instrument (selected by 0xE9 ii) replaces a single note with a short sub-program of timed steps, each setting a new volume and period. The note's pitch value chooses which member of the family plays — which drum, or which bass pitch.

The instrument sub-format:

bytesmeaning
0x2x ddconfig (tone/noise/envelope bits) + step duration dd frames
0x1xnoise period
V F · LLstep: volume V, 12-bit period (F = high nibble, LL = low byte)
0xFFend — channel returns to note mode

Instrument 1 is a 12-piece drum kit (note 0 = hi-hat, 2 = snare, 11 = bass drum…); instruments 4–5 are basses, one entry per pitch.

Instrument 5 — “Bass”, member C (note 0)

A single bass note is twelve volume/period steps: a loud attack that decays while the period drifts upward — a plucked-bass envelope built from register writes.

REUSELoops keep songs small

Songs are highly repetitive, so the bytecode never stores a repeated section twice. A family of branch commands (all beginning 0xFE) restructures the stream:

bytesmeaning
0xFE n  aabbloop back to address aabb, n times
0xFE FF aabbcall a shared passage at aabb (returns on 0xFF)
0xFE FE aabbunconditional jump (the song's outer loop)
0xFE 00switch the channel into sound-effect (direct) mode

Addresses are absolute 16-bit values, stored low byte first. Shared passages (choruses, bass figures) are written once and called from several points, the way a program calls a subroutine.

SFXSound effects: direct register steps

Sound effects run on the fourth channel and use the same step format as instruments — a linear list of config + period/volume + noise steps, executed a few frames apart. Some loop a short figure with the same 0xFE branch the music uses.

When an effect plays, it can take over a music channel by priority; the music data is saved and restored when the effect finishes.

Sfx “Pick up item” — decoded steps

A square tone stepped rapidly upward in pitch, then repeated at falling volume — the rising item-pickup chime, built step by step. (“Dog bark” loops one short figure twice.)
