Honesty notes:
- The **denominator** is parsed automatically from the `.asm` routine labels of each
  component's files (gameplay code only — MSX hardware init, the PSG/sound-driver internals,
  and copy-protection are not counted).

  and faithfulness varies — some routines are ported with documented divergences. Routines

## Offline tooling (not routine-counted)

These reimplement ROM **data decoding** in the C# export tools rather than porting game
logic to JS, so they are tracked qualitatively (the PSG/sound driver itself is out-of-scope).

- **Asset export (sprites / rooms / doors / alert icon / guard bullet)** — reimplemented in C# (MetalGearSpriteMover). RLE/sprite-attribute decoding + colour-combine compositing rebuilt offline to emit web/assets PNGs + atlases. Tracked qualitatively, not by routine count.
- **Audio export (SFX + music to WAV)** — reimplemented in C# (ThemeOfTaraPlayer). PSG emulation + the music/SFX driver rebuilt offline to render punch/door/alert/dead WAVs. The driver internals are out-of-scope for the gameplay denominator; tracked qualitatively here.

## What is covered, per component

### Guard (patrol, LOS, alert, chase, shoot, punch)
Core patrol + the FULL alert state machine + global alarm + reinforcements + punch-KO are ported. The alert AI (guardalert.asm) does chase (commit-then-re-aim, ~3/4 stop-and-shoot), wait-after-shot, obstacle detour, AND the RED-ALERT stand-off (SetGuardWalkAway/GuardWalkAwayShot statuses 3/4 via ChkNearPlayer max(|dx|,|dy|)<0x30), the RED-ALERT double-shot on resume + faster 0x0A cadence (SetGuardRndCounter), the status-4 init beat (GuardWaitChkAlert) that triggers the alarm, ChkGuardWater/MoveAwayExit (turn 180 off a door/lorry/chasm exit tile), and the lorry-interior dedup (InitGuardAlert2/3 + ChkDismissGuard via the Guard1/2/3ExitedLorry flags). LOS is now EXACT (chkdiscover.asm): the up/left-include-level facing asymmetry, the |dx|<8 / guard-|dy|<6 / camera-|dy|<4 sight bands, ChkViewObstacles see-through railings, and the noise/touch discovery (ListenShotsChkTouch/ChkDiscoverPlayer2-6 — a touch or an exploding shot wakes sleeping guards + sentinels). Reinforcements use the card-based budget (SetAlertMode2-4 NumRespawnGuards = highest card + 3; room 216 none; rooms 187/154/88-92 budget 10 via SetRespawnTime) spent on kills (DecRespawnGuards), with ChkAlarmEnd holding a red/camera alarm across rooms until the budget is spent. ### Enemy bullet actors (aimed shot + axis bullets)
InitShotToPlayer (ID_BULLET 0x3D: jetpacks, Hind D, lorry shooters, guard-switch) is ported as fireGuardBullet (hind-d change): the faithful quantized CalcShot2 aim at ShotSpeed 0x90 (QuadrantDegrees + the sin table, ported verbatim), player collision, the bullet sprite, and the pool cap; the Hind D compensates the torso-origin offset to fire from the actor's own XY like the ROM. Minor divergence: fireGuardBullet plays the shot SFX, which the ROM's InitShotToPlayer does not (SFX 5 belongs to InitBulletVert2). The InitBulletHor/InitBulletVert axis bullets (fixed ±5px/iteration along the axis toward screen centre + a random ±0x40 perpendicular jitter from the R register) are NOT implemented: their ROM spawners are the Shooter actor (rooms 88/90/91/206), the room-150 GuardSilencer pop-out shots, and Big Boss — the port substitutes aimed fireGuardBullet shots for Big Boss and approximates Shooter rooms with forced-alarm alert guards (undocumented divergences). InitBulletVert2's shared spawn tail (Moving=1, COLLISION_CFG=1, sprite 0x72, SFX 5) is effectively present via fireGuardBullet + updateBullets, hence partial.

### Snake movement + tile collision
