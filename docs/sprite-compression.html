CONTEXTThe constraint: limited ROM, many patterns

A sprite is a small bitmap the hardware draws as an independently movable object — a character, an enemy, a projectile. Each is a grid of pixels.

  • Metal Gear's sprites are 16×16 = 256 pixels each.
  • At full color (16-color palette = 4 bits/pixel) that is 128 bytes per sprite.
  • Snake alone requires dozens of frames (four facings × walk cycle × crawling, etc.), and the entire game must fit within a cartridge measured in kilobytes.

The pixel data is therefore stored in a compressed, compact form and expanded at draw time. Five distinct techniques do the work, covered below.

1 · 1-BITEncode shape only — one bit per pixel

The video hardware can render a sprite as a one-bit-per-pixel stencil: each pixel is either set or clear, with no color stored in the pattern itself.

This reduces a 16×16 sprite from 128 bytes (4 bits/pixel) to 32 bytes — a 4× reduction.

Right: a single 16×16 stencil taken directly from Snake's idle (facing-down) sprite in the ROM. Green marks a set bit; dark marks a clear bit.

The pattern carries no color yet; techniques 3–4 reintroduce it at low cost.

16×16 = 32 bytes · 1 bit/pixel

2 · TILESCompose 16×16 sprites from 8×8 tiles

The hardware's native pattern unit is 8×8. A 16×16 sprite is stored as four 8×8 tiles, in the order the hardware reads them:

TL · BL · TR · BR

Top-left, then bottom-left, then top-right, then bottom-right (column-major). Decomposing sprites into 8×8 tiles also lets a tile be reused across multiple sprites rather than stored repeatedly.

3 · ATTRStore color in a separate attribute

The stencil encodes shape and position. Color is supplied separately, by a compact attribute entry — one byte per hardware sprite.

Each entry holds four fields:

screen-Y, screen-X, pattern index, color

Because color is stored independently of the pixels, a single stencil can be displayed in several colors at no additional storage cost — a guard and a palette-swapped variant share one pattern.

Two entries sit at Y=E8 (upper), two at Y=F8 (lower), forming a 16×32 figure. The color values 4A / 4C are explained by the next technique.

4 · OROverlay two stencils to extend the palette

A single sprite displays one color. Additional colors are obtained — without storing full-color pixel data — by drawing two sprites at the same position and combining them.

Bit 6 of the color byte (hence 4A = 40 | 0A) sets the color-combination flag, instructing the hardware to merge this sprite with the one beneath it. Within the overlap, the two color indices are combined with a bitwise OR:

  • stencil A only → color 7 (teal)
  • stencil B only → color 10 (tan)
  • overlap of both → 7 OR 10 = 15 (dark outline)

Two 32-byte stencils (64 bytes) thus yield a three-color character, versus 128 bytes for an equivalent full-color sprite.

A + B (overlap) — Snake's upper half

5 · RLERun-length encode the pattern bytes

Finally, the pattern bytes are run-length encoded: a sequence of identical bytes is stored as a count and a value. Uniform margins and solid fills compress substantially.

Each chunk begins with a control byte B:

control Bmeaning
B < 80RUN — repeat the following byte B times
B ≥ 80COPY — copy the following (B−80) bytes verbatim
B = 00end of frame

Detailed sprites such as Snake consist mostly of COPY chunks (little repetition), so RLE yields only a small reduction here; graphics with large uniform regions (logos, backgrounds) compress considerably more.

Fixed reduction

The structural techniques reduce storage by a fixed ratio, independent of image content. One-bit encoding quarters a single pattern (128 → 32 B); the OR-overlay then adds a second pattern to obtain three colors, so a color character occupies 64 B — half the full-color size.

Variable reduction

RLE removes a content-dependent amount from the remainder — significant for uniform regions, minimal for high-detail patterns. Snake's four patterns total 128 B but store in 118 B (~8% here); graphics with large flat areas compress considerably more.

Together these techniques are pure compression and reuse, allowing a fully animated character to fit within the constraints of a 1987 cartridge.
